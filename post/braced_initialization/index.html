initializer[dcl.init]

Mon, Feb 20, 2017

According [8.2], C++’s resolution policy will prefer to consider any construct that could possibly be a declaration a declaration. thus the following statement

  Object object(int());

will be consider a function declaration, instead of creating a new object, even the code intends to create one. This is so-called C++'s most vexing parsing
